# 朝読み（asayomi）— オンライン読書会 運営支援ツール 仕様書

## 0. メタ情報

- バージョン: 0.1（ドラフト）
- 作成日: 2026-06-20
- 対象: 毎朝 ovice 上で実施するオンライン読書会の運営支援
- リポジトリ: `~/works/asayomi/`（独立 git リポジトリとして管理）

---

## 1. 概要・目的

毎朝 [ovice](https://www.ovice.com/) の共有スペース（音声・画面共有・チャット）で行うオンライン読書会の
**進行を補助する Web アプリ**。ovice の機能（音声・画面共有・チャット）はそのまま活かし、
本ツールは「読書箇所の共有」「感想の共有」「進捗・達成の可視化」を担う。

### 開催スタイル（現状）
- 開催: 毎朝
- 参加者: 約 10 名
- 利用環境: Mac 中心、ブラウザ
- 現在の本: GIVE & TAKE
- 1 回の流れ（約 30 分）:

```
① 本と今日読む範囲の説明
② 15 分の黙読（約 10 ページ進む）
③ 感想を音声でシェア（各 1 分 × 10 名）
④ 次回へ / 読了したら次の本を選ぶ
```

### 設計の大前提
- **キーボード入力は極力避ける。** 操作はアイコン選択・スライダー・ボタンが基本。
- ovice と併用する前提なので、機能を盛り込みすぎない（音声・チャットは ovice に任せる）。

## 2. 評価指標と設計方針

| 評価指標 | 設計方針 / 対応機能 |
|---|---|
| 本に対する理解が進む | 黙読タイマーで集中時間を確保。「今日の問い（お題）」を提示し、読みながら考える視点を与える。読書箇所を明示。 |
| 相互理解が進む | 感想シェア中、聞き手がリアクションアイコン（共感・なるほど・発見）をタップ。共感スライダーで温度感を共有。話者の順番をツールが表示。 |
| 読書が進む達成感 | 本全体に対する進捗バー。累計ページ・連続開催日数。終了時に「今日読んだ量」を可視化。 |
| 読了後の行動・生活変容 | 終了時に「明日試す一歩」をアイコンで選択。後日その振り返り。 |

---

## 3. 利用形態とフェーズ計画

### フェーズ 1（MVP）— 進行役 1 画面 + 画面共有
- 進行役だけがツールを操作し、ovice の**画面共有**で全員に見せる。
- バックエンド不要。データは進行役のブラウザ（localStorage）に保存し、JSON エクスポート可。
- 参加者の個別操作（リアクション等）は**この段階では進行役が代理**、または口頭/ovice チャットで受ける。

### フェーズ 2（拡張）— 各参加者が自分の端末で操作
- 参加者が各自 URL を開き、リアクション・スライダーを自分で操作。
- リアルタイム同期のためバックエンド（Supabase または Firebase を想定）を追加。
- フェーズ 1 のデータモデルをそのまま移行できるよう、**最初から後述のデータ構造で設計する**。

> 本仕様書はフェーズ 1 を実装対象とし、フェーズ 2 を見据えた構造を定義する。

## 4. 進行フローとツールの対応

| 進行ステップ | ツールの画面 / 機能 |
|---|---|
| ① 本と範囲の説明 | セッション開始画面: 本タイトル・今日読む範囲（開始〜終了ページをスライダーで設定）・今日の問い |
| ② 15 分黙読 | 黙読タイマー画面: 大きなカウントダウン、残り時間、開始/一時停止 |
| ③ 感想シェア | 感想シェア画面: 話者の順番表示、現在の話者ハイライト、1 分タイマー、リアクション集計 |
| ④ まとめ / 次へ | サマリー画面: 今日読んだページ数、本全体の進捗バー、連続開催日数、「明日の一歩」選択 |

## 5. 画面構成（フェーズ 1）

1. **ホーム / セッション準備**
   - 現在進行中の本（タイトル・著者・総ページ数）
   - 前回までの到達ページ（自動で「今日の開始ページ」に反映）
   - 今日読む範囲: 開始ページ（前回末を初期値）〜 終了ページをスライダー/＋−で設定
   - 今日の問い（任意・プリセットから選択）
   - 「黙読スタート」ボタン

2. **黙読タイマー**
   - 大きなカウントダウン（既定 15 分、＋/−で 5 分単位調整）
   - 開始 / 一時停止 / 終了
   - 終了 30 秒前の視覚的アラート（色変化）
   - 終了で自動的に感想シェア画面へ

3. **感想シェア**
   - 参加者リスト（順番）。ランダム並べ替えボタン。
   - 現在の話者をハイライト、1 分カウントダウン
   - 「次の人へ」ボタン
   - 聞き手リアクション: 共感 / なるほど / 発見 のアイコン（タップで＋1、集計表示）

4. **サマリー / クロージング**
   - 今日読んだページ数（終了 − 開始）
   - 本全体の進捗バー（到達ページ / 総ページ）
   - 連続開催日数（ストリーク）
   - 「明日の一歩」アイコン選択（例: 試す / 人に話す / 書き留める / もう一度読む）
   - セッションを保存（localStorage へ追記）
   - 読了時: 「次の本を選ぶ」導線（フェーズ 2 で投票化）

## 6. 機能仕様（詳細）

### 6.1 読書箇所の共有
- 開始ページ: 前回セッションの終了ページを初期値に自動設定。
- 終了ページ: スライダー or ＋/− ボタン。キーボード入力なし。
- 本の総ページ数は本の登録時に 1 度だけ設定（ここだけ数値入力を許容、以後は不要）。

### 6.2 黙読タイマー
- 既定 15 分。5 分単位で増減。
- マグニチュード大きめの表示（画面共有で全員が読める）。
- 残り 30 秒で配色変化（赤系）。
- 0 で通知音（任意・ミュート可）＋自動遷移。

### 6.3 感想シェア補助
- 参加者は事前登録（名前 or ニックネーム・アイコン）。10 名程度。
- 順番のランダム化。
- 各話者 1 分タイマー。「次の人へ」で進行。
- リアクション 3 種をタップで集計（フェーズ 1 は進行役が代理 / フェーズ 2 で各自）。

### 6.4 進捗・達成の可視化
- 進捗バー: 到達ページ / 総ページ（％）。
- 累計ページ数・連続開催日数（ストリーク）。
- 今日の読書量を数値とバーで表示。

### 6.5 行動・変容（「明日の一歩」）
- プリセットアイコンから 1 つ選択（自由記述は任意・基本は不要）。
- セッション記録に保存し、後日の振り返り（フェーズ 2）で参照。

## 7. データモデル（フェーズ 1 = localStorage、フェーズ 2 = DB に移行可能な形）

```jsonc
// books: 読んでいる本
{
  "id": "book_001",
  "title": "GIVE & TAKE",
  "author": "アダム・グラント",
  "totalPages": 376,
  "status": "reading",   // reading | finished | paused（新しい本に切り替えると読書中→pausedに自動移行）
  "startedAt": "2026-06-01",
  "finishedAt": null
}

// sessions: 1 回の開催
{
  "id": "session_20260620",
  "bookId": "book_001",
  "date": "2026-06-20",
  "startPage": 40,
  "endPage": 50,
  "question": "ギバーが消耗しない条件は？",   // 今日の問い（任意）
  "participants": ["p1", "p2", "..."],
  "reactions": { "empathy": 12, "insight": 8, "discovery": 5 },
  "nextSteps": [ { "participantId": "p1", "step": "try" } ],
  "durationMin": 15
}

// participants: 参加者
{
  "id": "p1",
  "name": "ゆう",
  "icon": "🦊",
  "joinedAt": "2026-06-01"
}
```

- localStorage キー: `asayomi.books` / `asayomi.sessions` / `asayomi.participants`
- JSON エクスポート/インポート機能を用意（バックアップ＆将来の DB 移行用）。
- フェーズ 2 では上記をそのままテーブル/コレクションへマッピング。

## 8. 技術スタック

### フェーズ 1
- **単一 HTML + バニラ JS（または軽量フレームワーク）** で動くフロントエンドのみ。
- 依存を最小化し、ブラウザで開くだけ / 簡易ホスティングで動く構成。
- 状態保存は localStorage。
- 対象ブラウザ: Mac の最新 Chrome / Safari。

### フェーズ 2（追加時）
- バックエンド: Supabase（Postgres + Realtime）または Firebase。
- 各参加者の端末から接続、リアルタイム同期。

---

## 9. 非機能・制約

- キーボード入力は本の総ページ登録など最小限に限定。日常操作はタップ/スライダー。
- 画面共有で読めるよう、文字・主要要素は大きめ。
- オフラインでも基本動作（localStorage）。
- 個人情報はニックネーム＋アイコンのみ。

---

## 10. フェーズ 2 拡張候補

- 各端末からのリアクション・共感スライダーのリアルタイム反映。
- セッション履歴ページ（カレンダー / 連続日数 / 本ごとの軌跡）。
- 「明日の一歩」の翌日振り返り（できた？をアイコンで回答）。
- 読了時の次の本を選ぶ投票機能。
- 感想の任意ボイス/テキストメモ添付。

---

## 11. 開発ステップ（フェーズ 1）

1. プロジェクト雛形（単一 HTML + JS + CSS）と git 初期化。
2. データモデル（localStorage ラッパ）と本・参加者の登録。
3. ホーム/準備画面（読書箇所スライダー・今日の問い）。
4. 黙読タイマー。
5. 感想シェア画面（順番・1 分タイマー・リアクション）。
6. サマリー画面（進捗バー・ストリーク・明日の一歩）＋セッション保存。
7. JSON エクスポート/インポート。
8. 通し動作確認（実際の朝会フローで試す）。
